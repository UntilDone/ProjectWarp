1. 데이터 ID
  1) ( 1~99 ) : Player 관련 요소
  2) ( 100 ~ 199 ) : 시계 및 시계 시스템
  3) ( 200 ~ 499 ) : 몬스터
  4) ( 500 ~ 999 ) : 카드
  5) ( 1000 ~ 1099 ) : 이벤트
  6) ( 1100 ~ 1100 ) : 버프, 디버프
  7) ( 1200 ~ 1299 ) : 맵
  8) ( 1300 ~ 1399 ) : 탐사대원
  9) ( 1400 ~ 1600 ) : 리워드 버튼
  10)( 1700 ~ 1800 ) : 후속 이벤트

2. 시연용 알파까지 구현되어야 하는 것
  1) splash Image
       [1] 팀 로고
       [2] 애니메이션
  
  2) 메인메뉴
       [1] 모든 메뉴 버튼만 구현
       [2] PlayButton 클릭 시 새로운 SaveData 생성하여 저장 : 달성
       [3} 이미 SaveData가 있는 상태라면 메인메뉴에 Resume 버튼과 Abandon 버튼을 활성화 시키고
            PlayButton을 비활성화 시킴
       [4] Abandon시 SaveData 삭제
       [5] PlayButton 클릭 시 맵 화면으로 이동, Resume 버튼 클릭시 SaveData를 Load하고 어떤 상황인지에 따라 맵 OR 전투화면으로 이동
  3) 맵 화면
       [1] 이벤트, 전투, 상점이 랜덤한 확률로 나오도록 구현
       [2] 나와있는 아이콘 또는 카드(로 맵을 처리하고있다면)를 클릭시 해당 화면으로 이동
       [3] 클릭시 SaveData에 Save하고 Resume시 해당 맵에서 시작하도록 설정

  4) 전투 맵
       [1] 각종 Data 연동
       [2] Deck_Data를 통해 Deck을 생성하고 시퀀스 대로 작동시키기
       [3] 카드 사용시 적 OR Player에게 적용하도록 하기
       [4] 사망시 현재까지 진행했던 요약 공개 및 메인메뉴로 가는 Button 만들고 클릭시 SaveData 삭제
  
  5) 이벤트 맵
       [1] eventData 연동 : 달성
       [2] 랜덤하게 뽑아서 진행 : 달성
       [3] 이벤트 버튼 클릭시 효과들을 적용하고 SaveData에 저장하며 여기서 종료를 하더라도 맵화면으로 돌아가게 만들기
       [4] 다음 Button을 만들어 맵화면으로 이동 : 달성

  6) 상점 맵
       [1] shopData 연동
       [2] 랜덤하게 뽑아서 진행
       [3] 아이템 및 카드 구매시 SaveData에 저장하고 여기서 종료를 하더라도 맵화면으로 돌아가게 만들기
       [4] 나가기 Button을 만들어 맵화면으로 이동
